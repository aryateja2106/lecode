# Security Policy

## Supported Versions

lecode is pre-release software (v0.0.x). Only the `main` branch receives security fixes until the first tagged release.

## Reporting a Vulnerability

Please **do not** file public GitHub issues for security vulnerabilities.

Report privately using one of the following:

- GitHub Security Advisories: https://github.com/aryateja2106/lecode/security/advisories/new
- Email: aryayt55@gmail.com with subject line `[lecode security]`

Please include:

- A description of the issue and its impact.
- Reproduction steps or a proof-of-concept.
- Affected versions or commit SHAs.
- Your disclosure timeline preference.

## Disclosure Process

1. We acknowledge your report within **5 business days**.
2. We investigate, reproduce, and assess severity.
3. We prepare a fix on a private branch and coordinate a release.
4. We publish a GitHub Security Advisory and credit reporters (unless you prefer anonymity).

Thank you for helping keep lecode and its users safe.
